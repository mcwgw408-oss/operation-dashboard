# Conversation Engine Design

日付: 2026-07-04

目的: First Agent の返事に対して、Sakura AI が自然にもう一言返すための設計を行う。

この文書は設計メモです。コード、README、localStorageデータ、Snapshot仕様は変更しません。

## 1. Conversation Engine の目的

Conversation Engine は、ユーザーの返事に対して、Sakura AI が短く自然な一言を返すためのレイヤーです。

目的:

- ユーザーの選択を受け止める。
- Recommendation を押しつけない。
- 次の行動を命令しない。
- 選択後の安心感を作る。
- First Agent の返事UIを、会話として自然に感じられるようにする。

Conversation Engine は、行動を実行するAgentではありません。

やらないこと:

- タスクを完了にする。
- 投稿する。
- メッセージを送る。
- 削除する。
- localStorageに自動保存する。
- 外部APIへ送信する。
- ユーザーの選択を訂正する。

Conversation Engine は、「返事に対して、もう一言添える」だけの小さな会話エンジンです。

## 2. First Agentとの関係

First Agent は、ユーザーの返事を受け取るレイヤーです。

Conversation Engine は、その返事に対して表示する一言を生成します。

関係:

```text
Recommendation
↓
User
↓
First Agent
↓
Conversation Engine
↓
Companion Response
```

First Agent が扱うもの:

- どの返事ボタンが押されたか。
- 任意メモがあるか。
- 返事が `やってみる`, `あとで`, `今日は休む`, `別のことをする` のどれか。

Conversation Engine が扱うもの:

- 返事に対する短い `Companion Response`。
- EnergyやIntentを踏まえた柔らかい言い換え。
- ユーザーの選択を尊重する言葉。

First Agent は入力を受け取る層であり、Conversation Engine は返す言葉を整える層です。

## 3. Recommendationとの関係

Recommendation は、Sakura AI Brain からユーザーへの提案です。

Conversation Engine は、Recommendation を作り直しません。

Conversation Engine がしてよいこと:

- Recommendation に対するユーザーの返事を受けて、一言返す。
- 「提案は候補であり、最終判断はユーザーにある」と感じられる文にする。
- Recommendation の強さを、返事に合わせてやわらげる。

Conversation Engine がしてはいけないこと:

- Recommendation の内容を勝手に変更する。
- 新しいPriorityを決める。
- 別の候補を自動で選ぶ。
- ユーザーの返事を否定する。

Recommendation は提案です。

Conversation Engine は、その提案を見たユーザーの選択に寄り添います。

## 4. 「もう一言」の役割

「もう一言」は、会話を進めるためではなく、選択を受け止めるためにあります。

役割:

- 返事が届いたことを伝える。
- ユーザーの選択を肯定する。
- 必要以上に次の行動を増やさない。
- 小さく始める、休む、保留する、別のことをする、どの選択も同じように尊重する。

例:

| 返事 | もう一言 |
|---|---|
| やってみる | いいですね。まず15分だけ始めてみましょう。 |
| あとで | 了解です。必要になったらまた一緒に考えましょう。 |
| 今日は休む | 今日は回復を優先しましょう。それも大切な選択です。 |
| 別のことをする | 了解です。今日は今の自分に合うことを選びましょう。 |

「もう一言」は短くします。

この一言は、ユーザーの選択を受け止める `Companion Response` として扱います。

長い説明、説得、再提案は避けます。

## 5. 会話の長さ

初回設計では、会話は1往復で終わります。

推奨:

```text
Recommendation
↓
User reply
↓
Companion Response
```

会話を長くしすぎると、ユーザーに負担がかかります。

初回実装では、Conversation Engine は追加質問をしません。

やらないこと:

- 「なぜですか？」と聞き返す。
- 理由入力を求める。
- 次の選択肢をさらに出す。
- 別のRecommendationを自動で生成する。
- 会話を続けるよう促す。

会話は、軽く受け止めて終わるのが基本です。

## 6. やさしい表現ルール

使いたい表現:

- 「了解です」
- 「大丈夫です」
- 「それも大切な選択です」
- 「今の自分に合うことを選びましょう」
- 「必要になったらまた一緒に考えましょう」
- 「まず少しだけ」
- 「完了まで行かなくて大丈夫です」
- 「今日は回復を優先しましょう」

文体のルール:

- 短くする。
- 命令形を避ける。
- 選択を肯定する。
- 行動を小さくする。
- 休む選択を同じ重みで扱う。
- 失敗や遅れを感じさせない。

良い例:

```text
了解です。今日は今の自分に合うことを選びましょう。
```

```text
今日は回復を優先しましょう。それも大切な選択です。
```

```text
必要になったらまた一緒に考えましょう。
```

## 7. 絶対にしない表現

Conversation Engine は、ユーザーを動かすために圧をかけてはいけません。

避ける表現:

- 「必ずやりましょう」
- 「今すぐ始めてください」
- 「やらないと遅れます」
- 「休んでいる場合ではありません」
- 「この提案に従うべきです」
- 「なぜやらないのですか」
- 「失敗しています」
- 「放置されています」
- 「せっかく提案したのに」

避ける振る舞い:

- 返事を否定する。
- 休む選択を下に見る。
- ユーザーを説得する。
- 返事に点数をつける。
- 次の行動を勝手に増やす。
- 自動保存や自動実行につなげる。

Conversation Engine は、ユーザーの意思を補助するための言葉だけを返します。

## 8. Energyとの関係

Energy は、Conversation Engine の文の強さを調整します。

Energy別の方針:

| Energy | 方針 | 例 |
|---|---|---|
| High Energy | 少し前向きに背中を押す | いいですね。流れがあるうちに少し進めましょう。 |
| Normal | 標準のやさしい応答 | 了解です。まず少しだけ始めてみましょう。 |
| Low Energy | 行動を小さくする | 完了まで行かなくて大丈夫です。触るだけで十分です。 |
| Recovery | 休む選択を強く尊重する | 今日は回復を優先しましょう。それも大切な選択です。 |

Energyが低いときは、行動量を増やす言葉を避けます。

EnergyがRecoveryのときは、`やってみる` が選ばれても、完了ではなく「触るだけ」「開くだけ」の言葉にします。

## 9. Intentとの関係

Intent は、「今日は何を大切にしたいか」を表します。

Conversation Engine は、ユーザーの返事とIntentの関係を見ます。

例:

| Intent | Reply | Conversation方針 |
|---|---|---|
| 記事を書く | やってみる | Intentと返事が一致しているため、軽く背中を押す |
| 記事を書く | あとで | 保留を尊重し、あとで戻れる言葉にする |
| 休む | 今日は休む | Intentと返事が一致しているため、休む選択を肯定する |
| 交流する | 別のことをする | 返事を優先し、Intentを押しつけない |

原則:

- Intentは参考にする。
- 最終判断はUser Replyを優先する。
- IntentとReplyが違っても、矛盾として責めない。
- 「今の選択」を尊重する。

## 10. 将来Memory Layerとどう繋がるか

初回実装では、Conversation Engine は保存しません。

Memory Layerとは、将来、明示保存が入ったあとに繋がります。

将来Memory Layerに渡せる情報:

- どの返事が選ばれたか。
- Energyがどうだったか。
- IntentとReplyが一致していたか。
- どのConversation文が表示されたか。
- ユーザーがその後保存を選んだか。

Memory Layerで将来できること:

- 休む選択が多い時期を把握する。
- Low Energy時に合う言葉を見つける。
- IntentとReplyのずれを長期的に観察する。
- ユーザーが受け取りやすい文体を参照する。

ただし、Memory Layerは勝手に学習しません。

保存、参照、学習は、すべて別設計で明示的に扱います。

## 11. Design Philosophy

Conversation は
ユーザーを動かすために存在するものではありません。

ユーザーが

「自分で選んだ」

と感じられるようにするためにあります。

Recommendation は提案です。

User Reply は選択です。

Conversation は
その選択を安心して受け止めるための言葉です。

Sakura AI は
人の代わりに決めるAIではありません。

人が自分で決められるよう、
そばで支えるAIを目指します。

そのため Conversation Engine は

- 短く
- やさしく
- 押しつけず
- 余白を残す

ことを基本方針とします。
