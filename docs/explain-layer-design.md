# Explain Layer Design

日付: 2026-07-04

目的: Sakura AI Brain が出した判断や提案について、AIがどう考えたのかをユーザーが理解できるようにする Explain Layer を設計する。

この文書は設計書です。コード、README、localStorageデータは変更しません。

## 1. Explain Layer の目的

Explain Layer の目的は、Sakura AI Brain の判断過程を、ユーザーが安心して確認できる形に整えることです。

Explain Layer は、AIが正しいことを証明するための層ではありません。

Explain Layer は、AIがどう考えたかを人が理解できるようにするための層です。

Explain Layer が扱うもの:

- どの情報を見たか。
- どの理由を重視したか。
- なぜその候補が上に来たか。
- なぜその提案文になったか。
- どの部分が仮の判断か。
- どの部分はまだ不確かか。

Explain Layer は、判断の正しさを押しつけるのではなく、ユーザーが自分で判断し直せる余白を残します。

## 2. Brainとの関係

Brain は、Snapshot、Dashboard、Intent Layer、各Laboの情報を読み取り、今日の状態を整理する中心です。

Explain Layer は、Brain が整理した内容を、ユーザーに伝わる説明へ変換します。

Brain が行うこと:

- 状態を整理する。
- 候補を見つける。
- 変化を検出する。
- Priority Engine や Recommendation Engine に渡す材料を作る。

Explain Layer が行うこと:

- Brain が見た材料を説明する。
- Brain が重視した観点を説明する。
- Brain の判断がどこまで確かかを伝える。
- ユーザーが納得できない場合に見直せる形で表示する。

Explain Layer は Brain の外側にあり、Brain の判断を上書きしません。

Explain Layer は、Brain の思考を人に見える形へ整える補助レイヤーです。

## 3. Priority Engineとの関係

Priority Engine は、候補を集め、Priority Score、理由、Energy Modifier、Momentum Modifier を使って優先候補を出します。

Explain Layer は、その計算結果をそのまま点数だけで見せるのではなく、自然な説明へ変換します。

説明対象:

- 候補として選ばれた理由。
- スコアに影響した要素。
- Energy が提案に与えた影響。
- Momentum が説明文に与えた影響。
- 他の候補より優先された理由。

例:

```text
今日の最優先候補は「記事を書く」です。
発酵中アイデアと執筆中の記事があり、最近の流れも続いているため、今つなげる価値が高いと判断しました。
ただし、エネルギーが低い日は15分だけ試す形がよさそうです。
```

Explain Layer は、Priority Score を絶対的な答えとして扱いません。

Score は説明の材料であり、最終判断はユーザーに残します。

## 4. Recommendation Engineとの関係

Recommendation Engine は、Priority Engine の結果をもとに、自然な提案文を作ります。

Explain Layer は、Recommendation がなぜその言い方になったのかを補足します。

Recommendation Engine が行うこと:

- 優先候補を提案文に変える。
- 命令ではなく提案として表現する。
- Energy や Momentum に合わせて文体を調整する。
- 候補がない場合の自然な提案を出す。

Explain Layer が行うこと:

- その提案になった背景を説明する。
- 強くすすめているのか、軽く提案しているのかを明確にする。
- 不確かな場合は不確かさを伝える。
- 休む提案にも理由を添える。

例:

```text
この提案は、作業候補があることと、最近の流れが続いていることをもとに作られています。
一方で、エネルギーが低めの場合は、長時間ではなく短い着手をすすめる表現にしています。
```

Explain Layer は、Recommendation のやさしさを壊さず、必要なときだけ理由を添える役割を持ちます。

## 5. 説明する内容

Explain Layer が説明する内容は、ユーザーが判断を理解するために必要な範囲に限定します。

説明するもの:

- 見たデータの種類。
- 優先候補にした理由。
- スコアに影響した主な要素。
- Energy の影響。
- Momentum の影響。
- Intent Layer から読み取った意思。
- 候補がない場合の理由。
- 提案が弱め、強め、休息寄りになった理由。
- 不確実な点。

説明例:

```text
今日は「執筆中の記事」と「発酵中アイデア」があるため、書くことを候補にしました。
ただし、回復ログでは疲れが見えるため、短く始める提案にしています。
```

説明は、長すぎないことを基本にします。

ユーザーが必要なときに詳しく見られる構成が望ましいです。

## 6. 説明しない内容

Explain Layer は、すべてを説明しようとしません。

説明しないもの:

- ユーザーを評価する言葉。
- 正解、不正解の断定。
- 心理状態の決めつけ。
- 医療的、法的、専門的な診断。
- 保存していない情報を見たように見せる表現。
- AIが確信していないことを確定的に言う表現。
- 複雑すぎる内部計算の詳細。
- ユーザーが求めていない長い分析。

避ける例:

```text
あなたは最近やる気がないので、この作業を優先すべきです。
```

自然な表現:

```text
最近の記録では少し疲れが見えるため、今日は短く始められる形が合いそうです。
```

Explain Layer は、ユーザーを説明対象にしすぎません。

説明するのは、AIの考え方です。

## 7. やさしい説明ルール

Explain Layer の説明は、短く、やさしく、押しつけない言葉にします。

基本ルール:

- 断定しすぎない。
- 理由を1から3個に絞る。
- 点数よりも意味を説明する。
- ユーザーの状態を決めつけない。
- 「だからやるべき」ではなく「そのため候補にしました」と言う。
- 休む選択も同じ重さで尊重する。
- 不確かな場合は「かもしれません」「見えます」と表現する。
- ユーザーが違うと感じる余白を残す。

推奨表現:

- 「この候補が上に来た理由は、主にこの2つです。」
- 「今日は無理に進めるより、短く試す提案にしています。」
- 「記録上はこの流れが見えていますが、違和感があれば無視して大丈夫です。」
- 「休む提案になったのは、回復を優先する情報が見えているためです。」

避ける表現:

- 「これが最適です。」
- 「必ずこれをしてください。」
- 「あなたはこういう人です。」
- 「AIの判断に従ってください。」

## 8. Userとの関係

User は、Sakura AI Brain の判断を受け取る主体ではなく、最終的に選ぶ主体です。

Explain Layer は、User が判断を理解し、必要なら選び直せるようにするためにあります。

User に対して提供するもの:

- 理由の見える化。
- 不確実性の明示。
- 選び直す余白。
- 休む選択の尊重。
- AIの提案を距離を置いて見られる安心感。

Explain Layer は、User に納得を強制しません。

Explain Layer は、User が「なぜそう言われたのか」を確認できる窓です。

そのため、説明はいつでも閉じられる、読み飛ばせる、軽く扱える形が望ましいです。

## 9. 将来Memoryとの関係

将来 Memory Layer が導入された場合、Explain Layer は Memory を使った判断も説明できる必要があります。

ただし、Memory はユーザーのために存在します。

Memory を説明に使う場合は、何を参照したかを明示します。

例:

```text
過去の記録では、疲れている日は短い作業のほうが続きやすい傾向があるため、今日は15分だけの提案にしています。
```

Memory と接続する場合のルール:

- Memory参照は明示する。
- 保存していない記憶を参照したように言わない。
- ユーザーが削除したMemoryは使わない。
- MemoryのON/OFF設定を尊重する。
- 個人傾向を断定しない。
- 「過去の記録では」と前置きする。
- 必要以上に過去を持ち出さない。

Explain Layer は、Memory の存在を透明にします。

ユーザーが「何をもとにそう言ったのか」を確認できることが重要です。

## 10. Design Philosophy

Explain Layer は、AIが正しいことを証明するための層ではありません。

Explain Layer は、AIがどう考えたかを人が理解できるようにするための層です。

Sakura AI Brain は、ユーザーの代わりに決めるAIではありません。

Sakura AI Brain は、ユーザーが自分で決められるよう、そばで支えるAIです。

そのため Explain Layer は、判断を押しつけるために存在しません。

Explain Layer は、判断の背景を開き、ユーザーが安心して受け取る、疑う、選び直すことを支えます。

説明は、説得ではありません。

説明は、透明性です。

説明は、ユーザーの選択を奪うためではなく、選択を取り戻すためにあります。

Sakura AI にとって大切なのは、
AIが正しいと見せることではありません。

人が、
「なぜそう提案されたのか」
を理解し、
「それでも自分で選んでいい」
と思えることです。

Explain Layer は、その余白を守るためのレイヤーです。

Explain Layer は、
AIがブラックボックスにならないための約束でもあります。
