# さくらAI だいじょうぶノート

忘れたときに開くノート。設計書ではなく、使う人（さくらさん）のための説明書。
むずかしいことは書いていません。ぜんぶ、実際に聞かれた質問から作りました。

作成日: 2026-07-10（Fable 5とさくらの会話より）

---

## まず、これだけ覚えていれば大丈夫

- **空欄は正常。** 書かない日、押さない日があっても、何も壊れません
- **忘れたらAIに聞く。** 「バックアップのやり方忘れた」で通じます。このノートを読んで案内してくれます
- **忘れたら台無しになるものは、この画面にひとつもありません**

---

## 朝のつかい方（5分の儀式）

1. 画面を開く。「今日のひとこと」を流し読み
2. **Cockpit（今日の意図）の前でひと呼吸。** 今日の自分を確認して、方向があれば一言（例:「noteを少し進めたい」）。**浮かばなければ空欄のまま進んでいい**。空欄は「今日はさくらに任せる」という選択
3. 提案を読む。意図を書いた日は、自分の言葉が復唱されているはず
4. 絵文字ボタンを1つ押す（😊やってみる／🤔あとで／😴今日は休む／✨別のことをする）。**正解はない。今の気持ちに一番近いもの**。「休む」を押した朝も、さくらはちゃんと学びます
5. 返答を読んで、「自然だった」か「違和感」を押す
6. 状態メモにひとこと（「ねむい」だけでもOK。浮かばなければ空欄で終わり)

## 4つの「書く場所」の使い分け

| 場所 | 誰のため | いつまで | 例 |
|---|---|---|---|
| Cockpit(今日の意図) | さくらへ | 今日だけ | 「今日はnoteを進めたい」 |
| 今日さくらに覚えてほしいこと | さくらへ | ずっと(提案の材料) | 「木曜は訪問看護の日」 |
| 学びの蓄積 | 自分へ | ずっと(日付つきの気づき) | 「書き出しは体験から入ると筆が進む」 |
| 残るメモ | 自分へ | ずっと(日付と関係ない付箋) | 「noteヘッダー画像は1280×670」 |

- 学びの蓄積は、**書いたら「追加」ボタンを押す**とリストに残る（押し忘れると保存されない）
- 学びの中で「さくらにも知っていてほしい」ものだけ、リストの**「記憶へ」ボタン**で昇格できる。毎回ではなく、特別なものだけ
- 書く場所を間違えても、何も壊れません

## さくらの記憶とは

さくらAIが「さくらさんのこと」を覚えておくタンス。
- 入れ方: 「覚えてほしいこと」に書く／学びの「記憶へ」ボタン
- 使われ方: 毎朝の提案の材料になる（覚えるほど提案がさくらさん仕様になる）
- 見る・整える: 「ふだんは閉じているもの」の中の**さくらの記憶一覧**（検索と「忘れる」操作あり）

## 「ふだんは閉じているもの」の中身

閉じているのが正しい状態。開けるのは用事があるときだけ。
- **さくらパネル**: さくらの今日をJSONにして、AI運用部屋に持っていくとき
- **データ**: 過去データの検索と、バックアップ（下の週1習慣で使う）
- 体調分析: 傾向を振り返りたい週に
- さくらのなかみ: 開発者向け。普段はOFFのまま
- 育てるプロジェクト: 役目を終えた古い機能。開かなくてよい

## 週に1回だけの習慣: バックアップ（1分）

1. 「ふだんは閉じているもの」→ データ →**「バックアップを書き出し」を押す**（JSONファイルがダウンロードされる）
2. **ファイルを「さくらバックアップ」フォルダへ移す**
   - 場所: OneDrive → デスクトップ → working → **さくらバックアップ**（operation-dashboardの中ではなく、隣）
   - やり方: エクスプローラーを2つ開いて、ダウンロードのファイルをドラッグ。または右クリック→切り取り→貼り付け
3. 古いファイルは2〜3個だけ残して、それより前は消してよい

- 曜日を決めるのがコツ（訪問看護の日とセットなど)
- 1回忘れても大丈夫。先週の分が守ってくれます
- 慣れてきたら月1回、「読み込み」で復元の練習をするとさらに安心

## 困ったとき

- **画面が真っ白・時刻が出ない**: 何も操作せず、AIに「画面が出ない」と伝える。**データは消えていません**（画面が読み込む前に転んでいるだけ）
- **直したはずが直っていない**: Codexに頼みごとをしたら、**必ず「コミットhash」を受け取る**。hashのない「直しました」は信用しない。hashをAIに見せれば、本当に直っているか確認してくれます
- **何かおかしい・不安**: 週1バックアップがあるかぎり、だいたいのことは戻せます

## AIとの付き合い方（引き継ぎメモ)

- 設計・レビューの相談: この docs フォルダの review-playbook / design-decisions / open-questions を読んだAIなら、誰でも引き継げます
- 実装の頼みごと: Codexへ。小さな修正でもコミットhashをもらう
- 週1回、15分の振り返り: 「今週いちばん覚えている朝」をひとつ話すだけで十分
